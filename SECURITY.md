# Политика безопасности

## Сообщить об уязвимости

Нашли проблему безопасности (утёкший секрет в snippet, вредоносный внешний скилл,
дыру в скриптах установки)? Напишите приватно на **dv.vostrikov@gmail.com** —
не открывайте публичный issue до согласования.

Опишите: затронутый файл/скилл, шаги воспроизведения, потенциальное воздействие.
Ответ — в течение нескольких дней; исправление и раскрытие — после согласования.

## Модель доверия

Реестр держит провайдер-нейтральные `SKILL.md` + `snippets/`. Доверие к контенту
выстроено по приоритету: **проект > источник > реестр**. Проект всегда может
переопределить навязанное реестром поведение, источник (например Laravel Boost) —
ядро реестра.

### Внешние (imported) скиллы

Каждый внешний скилл сопровождается `upstream.json` рядом с `SKILL.md`:

- **`source` / `files[].url`** — откуда взят файл (github / http / file).
- **`files[].sha256`** — хеш точной версии, импортированной в реестр. При
  синхронизации (`scripts/update-upstreams.sh`) новый апстрим скачивается, хеш
  пересчитывается и сравнивается с зафиксированным.
- **`strategy`**:
  - `notify` (по умолчанию) — расхождение хеша не применяется автоматически,
    а попадает в PR с диффом для ручного ревью (workflow `upstream-sync.yml`).
  - `replace` — доверенный источник, можно обновлять автоматически.
- **`license`** — лицензия источника сохраняется; реестр её не перетирает.

Это значит: внешний контент **не исполняется и не обновляется молча** — изменения
видны как diff в PR до того, как попадут в чьи-либо проекты.

### Нет секретов в snippets

Сниппеты извлекаются из реальных проектов сканером (`scripts/scan-skills.sh
--extract`), который анонимизирует пути и неймспейсы. Перед мёрджем
PR-чеклист (`.github/PULL_REQUEST_TEMPLATE.md`) явно требует:

- нет секретов/кредов в сниппетах;
- неймспейсы анонимизированы.

`scripts/validate.sh` проверяет структуру (frontmatter, манифесты, граф
зависимостей) на каждом PR; тесты гоняются на матрице Python (`test.yml`).

## Установка и распространение

CLI ставится симлинком на `bin/swissknifeman` (`./install.sh`) — внешних
зависимостей нет, только stdlib Python 3. Скрипты установки не выполняют
сетевых запросов и не требуют root. Состояние подключённых проектов хранится
локально в `~/.swissknifeman/projects.json`.

## Поддерживаемые версии

Поддерживается последний релиз на `main`. Откат: `git checkout <tag>` в
репозитории реестра + повторный `./install.sh`.
