# Task Brief Template

## Когда активировать

Используйте этот skill, когда задача неоднозначна или затрагивает несколько подсистем.

## Шаблон постановки

```text
Контекст:
- Коротко: где проблема/инициатива и почему это важно.

Цель:
- Какой результат должен получить пользователь/бизнес.

Границы:
- Что входит в задачу.
- Что явно не входит.

Инварианты:
- Какие правила и контракты нельзя нарушать.

Затронутые слои:
- backend / frontend / policies / routes / docs / tests.

Ожидаемое поведение:
- До изменения:
- После изменения:

Критерии приемки:
- [ ] ...
- [ ] ...

Минимальная верификация:
- Какие тесты/проверки обязательно запустить.

Ограничения:
- Срок, риски, legacy-ограничения, обратная совместимость.
```

## Мини-чеклист качества постановки

- Цель описывает результат, а не только действия.
- Есть явные не-цели (out of scope).
- Перечислены инварианты домена и безопасности.
- Указан минимальный test-gate.
- Зафиксирован формат финального отчета по задаче.
