# Поддержка документации после задач

Внутренний скилл (не экспортируется). Активируется как **шаг-оценка** в конце
задачи, не обязательно как правка. CHANGELOG — не сюда
(это `release-discipline`).

## Когда активировать

В конце каждой завершённой задачи, если в корне репозитория есть `docs/`.

## Критерии «достойно документации»

**ДА, обновлять docs/:**

- Изменился процесс (контрибуция, валидация, релизы, синк апстримов).
- Появилась/изменилась возможность установки или дистрибуции
  (флаги install.sh, команды CLI, профили, каналы).
- Скрипт получил новое пользовательское поведение или флаги.
- Изменилась схема: skills.json, upstream.json, frontmatter
  (SKILL_TEMPLATE), buckets.json, `.swissknife.json`, projects.json.
- Приземлился пункт roadmap → обновить и `docs/roadmap.md`.

**НЕТ, docs не трогать:**

- Контент отдельных скиллов (skills.json и graph.md регенерируются сами).
- Опечатки, форматирование, рефакторинг без изменения поведения.
- Обновления зависимостей, тестовые фикстуры.

## Алгоритм

1. Перечислить, что изменила задача (scope git diff).
2. Прогнать дерево критериев выше.
3. **НЕТ** → явно зафиксировать в ответе: «docs не требуются: <причина>» — и стоп.
4. **ДА** → обновить существующую страницу (расширение предпочтительнее новой
   страницы); новая страница — только для действительно новой темы, тогда же
   добавить её в sidebar (`docs/.vitepress/config.mts`).
5. Сомневаешься, заслуживает ли тема страницы — предложи пользователю,
   не пиши молча.

## Карта docs/ по темам

- Установка/каналы/CLI: `docs/guide/installation.md`, `docs/guide/cli.md`,
  `docs/guide/profiles.md`, `docs/guide/vendor-skills.md`.
- Реестр и схемы: `docs/guide/registry.md`, `docs/guide/skill-anatomy.md`.
- Внешние скиллы: `docs/guide/upstream-sync.md`.
- Создание скиллов: `docs/guide/creating-skills.md`.
- Адаптеры/сканер/CI: `docs/guide/adapter-deltas.md`, `docs/guide/scanner.md`,
  `docs/guide/ci.md`.
- Провайдеры: `docs/workflows/*`, `docs/setup/*`.
- `docs/guide/graph.md` — **генерируется, не редактировать**.
- `docs/roadmap.md`, `docs/routing.md`, `docs/boost-compatibility.md`,
  `docs/index.md`.

## Чеклист

- [ ] Решение зафиксировано явно (обновил / «docs не требуются: причина»).
- [ ] Выбрана правильная страница; новая страница добавлена в sidebar.
- [ ] `graph.md` не редактировался руками.
- [ ] Roadmap обновлён, если приземлилась запланированная фича.
