# Writing Style

## Контекст

Комментарии и сообщения коммитов читают люди — а агент по умолчанию пишет
канцеляритом и пересказывает код. Этот скилл задаёт живой, предметный стиль:
комментарий объясняет **почему**, коммит описывает завершённое действие с
причиной. Формат Conventional Commits (типы, scope, footer) — в `git-commit-rules`;
здесь — про язык и тон.

## Когда активировать

При написании комментариев в коде и сообщений коммитов (в дополнение к
`git-commit-rules`, который отвечает за формат и хук commitlint).

## Комментарии в коде

- Только когда без комментария неочевидно **почему** так сделано — не дублируй
  то, что уже видно из кода.
- Коротко, настоящим/прошедшим временем: «проверяем», «ограничили», «оставили».
- Избегай «выполняет / осуществляет / производит» и пересказа очевидного.

```php
// Хорошо — объясняет причину:
// Ограничили выборку активными записями, чтобы не тянуть архив в публичный список.

// Плохо — пересказ кода канцеляритом:
// Данный код осуществляет выборку записей.
```

## Сообщения коммитов

- Заголовок — **завершённое действие**: «добавили», «исправили», «убрали»
  (не инфинитив «добавить/исправить»).
- В body — причина и эффект в 1–2 коротких предложениях.
- Формат (тип, обязательный scope, breaking) — по `git-commit-rules`.

```text
chore(deps): добавили rector-laravel в dev-зависимости

Нужен для наборов RectorLaravel в rector.php; lock обновлён через composer.
```

## Чеклист качества

- [ ] Комментарии объясняют «почему», не пересказывают код; их нет там, где и так очевидно.
- [ ] Нет канцелярита («осуществляет/производит/данный код»).
- [ ] Заголовок коммита — завершённое действие, не инфинитив.
- [ ] В body коммита — причина и эффект, кратко.
